# LinkedIn Scraper — Automated Prospecting Suite

A full-stack LinkedIn automation platform with a browser dashboard, REST API, email campaign system, and AI-powered CV generation. Runs on Windows, macOS, and Linux with a single launcher file.

---

## Features

| Feature | Description |
|---|---|
| **Profile Scraping** | Scrape LinkedIn group members, search results, or Google-sourced profiles |
| **Profile Enrichment** | Visit each profile to extract email, company, title, about — then generate likely email addresses |
| **Connection Campaigns** | Send single or mass connection requests from CSV |
| **Group Messaging** | Automated outreach messages to group members |
| **Email Campaigns** | Full SMTP campaign manager: create, schedule, send, retry, rotate accounts |
| **AI CV Generation** | Generate LaTeX CVs from profile data using Groq (free tier) |
| **Browser Dashboard** | Web UI to control everything without writing code |
| **REST API** | Full FastAPI backend — every feature accessible programmatically |

---

## Quick Start

### Windows — double-click or run in terminal

```batch
start.bat
```

### Linux / macOS — first time only

```bash
chmod +x start.sh
./start.sh
```

The launcher automatically:
1. Detects or installs Python 3.8+
2. Creates an isolated virtual environment
3. Installs all dependencies
4. Runs a 3-step configuration wizard (LinkedIn credentials, API key, Groq AI)
5. Launches the server and opens the browser dashboard

**Subsequent launches** skip setup and go straight to the server.

| Flag | Effect |
|---|---|
| `--reconfigure` / `--Reconfigure` | Re-run the setup wizard |
| `--update` / `--Update` | Reinstall/upgrade Python packages |
| `--help` / `--Help` | Show usage |

## Manual Installation

```bash
# 1. Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate        # Linux/macOS
venv\Scripts\activate           # Windows

# 2. Install dependencies
pip install -r requirements.txt

# 3. Copy and fill in the environment file
cp .env.example .env
# Edit .env with your LinkedIn credentials and API key

# 4. Launch the server
python -m uvicorn api.app:app --host localhost --port 8000

# 5. Open the dashboard
# http://localhost:8000
```

## Configuration

All settings live in `.env`. Copy `.env.example` to get started. The most important variables:

```env
# Required for scraping
LINKEDIN_EMAIL=your-email@example.com
LINKEDIN_PASSWORD=your-password

# Required for API access (auto-generated by start.bat / start.sh)
API_KEY=your-40-char-key

# Optional — enables CV generation
GROQ_API_KEY=your-groq-key

# Server
API_PORT=8000
```

> **Browser dashboard**: No API key needed — localhost requests bypass auth automatically.
> **Remote/programmatic access**: Pass header `X-API-Key: your-key`.

See `.env.example` for all available settings with descriptions.

## Email Campaign Workflow

1. **Add an SMTP account** (Accounts tab) — Gmail, Outlook, or custom
2. **Create a campaign** (Email tab) — name, subject, body with `{first_name}` / `{company}` / `{email}` variables
3. **Prepare** — system matches campaign to enriched profiles and builds the email queue
4. **Preview** — verify the rendered output before sending
5. **Send** — sends immediately, or use **Schedule** for daily recurring sends
6. **Monitor** — track pending / sent / failed per campaign

Supported template variables: `{first_name}`, `{last_name}`, `{full_name}`, `{email}`, `{company}`, `{title}`, `{location}`

## Profile Enrichment

Enrichment visits each LinkedIn profile to extract:
- Full name, title, company, location
- About section
- Work experience (job titles, companies, dates)
- Education

Then it:
1. Searches Google to find the company's domain
2. Generates likely email addresses in 4 formats:
   - `first.last@domain.com`
   - `first@domain.com`
   - `firstlast@domain.com`
   - `flast@domain.com`

Results saved to SQLite and exportable as CSV/Excel.

### Key Design Decisions

- **Background jobs** — long-running scrapes run in background threads; the API returns a `job_id` immediately and the client polls `/api/jobs/{job_id}`
- **Single SQLite file** — all data in one file at `data/db/linkedin_scraper.db`; WAL mode for concurrent reads
- **Localhost auth bypass** — the API key check is skipped for `127.0.0.1` / `::1` / `localhost`, so the browser dashboard works without credentials
- **Profile reuse** — `DriverManager` can launch Chrome with your existing browser profile, so scraping reuses your LinkedIn session without re-login
- **Component isolation** — all Selenium element interactions live in `components/` so routes and services never touch XPath directly

## Authentication

The scraper supports two authentication modes:

### Browser Profile (recommended)

Go to the **Auth** tab in the dashboard and click **Detect Profiles**. Select the Chrome profile where you are already logged into LinkedIn. The scraper will reuse that session — no credentials stored, no re-login.

### Credentials

Set `LINKEDIN_EMAIL` and `LINKEDIN_PASSWORD` in `.env`. The scraper will log in automatically using Selenium, save cookies to `.auth/cookies.pkl`, and reuse them on future runs.

> Use a dedicated LinkedIn test account. Automation from your main account risks temporary restriction.

## AI Features (Groq)

CV generation uses the Groq free API. Get a key at https://console.groq.com (no credit card required).

Set `GROQ_API_KEY` in `.env`. The system uses `llama-3.3-70b-versatile` by default with automatic fallback to other free-tier models if rate-limited.

## Data & Privacy

- All data is stored locally in `data/db/linkedin_scraper.db`
- Authentication cookies in `.auth/cookies.pkl`
- Nothing is sent to external servers except:
  - LinkedIn (scraping)
  - Groq API (CV generation, only when requested)
  - SMTP server (email sending)

The `data/` and `.auth/` directories are git-ignored by default.

## Troubleshooting

**Server doesn't start**
- Check `data/logs/server.log` for the error
- Make sure port 8000 is not in use — set `API_PORT=8001` in `.env`

**Chrome driver error**
- Update Chrome to the latest version
- The launcher auto-installs a matching ChromeDriver via `webdriver-manager`

**LinkedIn blocks the scraper**
- Add longer delays: `CONN_MIN_DELAY=30` in `.env`
- Use a dedicated test account
- Use the browser profile method (Auth tab) instead of credentials

**Gmail authentication fails**
- Use an App Password, not your regular password
- Enable 2FA on your Google account first, then generate an App Password

**CV generation fails**
- Check your `GROQ_API_KEY` in `.env`
- The free tier has rate limits — the system retries automatically
- PDF output requires `pdflatex` installed separately
